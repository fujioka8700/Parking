## 異常コードシートのデータ構造

### ヘッダー行（3行目）
| 列 | 列名 | 説明 |
|---|---|---|
| A | (Mコード) | 故障コード（M32, M33など） |
| B | コード | 表示コード（1, 2, 3など） |
| C | 名称 | 故障名称 |
| D | 内容 | 故障内容（詳細説明） |
| E | チェック項目 | 対応方法・チェック項目 |

### データ統計
- **総故障コード数**: 120個
- **有効な故障コード（欠番以外）**: 69個
- **欠番**: 51個

### データ例

#### 故障コード M32
- **表示コード**: 1
- **名称**: 非常停止
- **内容**: 非常停止釦ON、脱出扉開のいずれかを検知（動作モード10発行時を除く）
- **チェック項目**: 非常停止ボタンが押されていないか　また、非常脱出扉が開いていないか　ボタンの接点不良はないか　配線（断線・端子での抜け等）の確認

#### 故障コード M33
- **表示コード**: 2
- **名称**: 場内安全確認センサ動作
- **内容**: 自動モード切替え時に、場内安全確認センサーON検知
- **チェック項目**: センサの動作状態、光電センサの場合は光軸のチェック及び配線（断線・端子での抜け等）の確認　感度のチェック

## データマッピング

エクセルファイルからデータベースへのマッピング：

| エクセル列 | データベースフィールド | 型 | 説明 |
|---|---|---|---|
| A列（Mコード） | faultCode | String | 故障コード（M32, M33など） |
| B列（コード） | displayCode | String | 表示コード（1, 2, 3など） |
| C列（名称） | faultName | String | 故障名称 |
| D列（内容） | faultContent | String | 故障内容 |
| E列（チェック項目） | solution | String | 対応方法・チェック項目 |

## 注意事項

1. **欠番の処理**: 「欠番」という名称の故障コードは実際には存在しないため、アプリでは除外するか、特別な扱いが必要
2. **Mコードと表示コード**: 両方のコードで検索できるようにする必要がある
3. **データの完全性**: 一部の故障コードには「内容」や「チェック項目」が空の場合がある
